## 1. 测试策略概览

| 测试层级   | 覆盖范围                        | 工具/框架                            | 优先级 |
| ---------- | ------------------------------- | ------------------------------------ | ------ |
| 单元测试   | 各模块核心服务类                | JUnit 5 + Mockito + Spring Boot Test | P0     |
| 集成测试   | 组件间通信链路（WS/REST/Redis） | Spring Boot Test + Testcontainers    | P0     |
| 端到端测试 | 全链路用户场景                  | 手动 + Selenium/Playwright           | P0     |
| 安全测试   | 认证鉴权、注入防护              | 手动 + OWASP ZAP                     | P0     |
| 性能测试   | 消息吞吐、WebSocket 并发        | JMeter / Gatling                     | P1     |
| 高可用测试 | 多实例路由、故障恢复            | 手动测试                             | P1     |
| 回归测试   | IM 原有功能不受影响             | 手动测试清单                         | P0     |

### 2.1 AkSkAuthService — 签名验证

#### TC-GW-U01：签名验证通过

| 项目     | 内容                                                                                                                                                              |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 模块     | `AkSkAuthService.verify()`                                                                                                                                        |
| 前置条件 | `ak_sk_credential` 表存在 AK="test_ak"、SK="test_sk"、status=ACTIVE 的记录；Redis 可用                                                                            |
| 测试步骤 | 1. 生成当前时间戳 `ts`<br>2. 生成随机 nonce<br>3. 计算签名 `HMAC-SHA256(SK, AK + ts + nonce)` 并 Base64 编码<br>4. 调用 `verify("test_ak", ts, nonce, signature)` |
| 预期结果 | 返回关联的 userId；Redis 中 `gw:auth:nonce:{nonce}` 键被设置，TTL ≤ 300s                                                                                          |

#### TC-GW-U02：签名不匹配

| 项目     | 内容                                                                                |
| -------- | ----------------------------------------------------------------------------------- |
| 模块     | `AkSkAuthService.verify()`                                                          |

| 测试步骤 | 1. 用错误 SK "wrong_sk" 计算签名<br>2. 调用 `verify(ak, ts, nonce, wrongSignature)` |
| 预期结果 | 返回 `null`；Redis 中 nonce 键保留（已存入Redis，等待过期）                         |

#### TC-GW-U03：时间戳超窗（>5 分钟）

| 模块     | `AkSkAuthService.verify()`                                  |

#### TC-GW-U04：时间戳格式非法
