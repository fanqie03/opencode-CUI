> 版本：V2 | 日期：2026-03-02 | 关联：`00-方案设计主文档.md`

---

## 1. 阶段拆分

本项目分为 **2 个里程碑（Phase）**，仅实现 OpenCode 本地模式。

| 阶段 | 名称 | 目标 | 验收标准 |
|---|---|---|---|
| **Phase 1** | 核心链路（端到端打通） | 从 "/" 触发到流式对话全链路可用 | AC-1 ~ AC-4, AC-6 ~ AC-9 |
| **Phase 2** | 发送到 IM + 体验打磨 | 选文发送、HA、体验优化 | AC-5 |

## 6. 风险检查点

### 6.1 Phase 1 风险检查点

| 检查点 | 时间节点 | 检查内容 | 应对措施 |
|---|---|---|---|
| **CK-1** | GW-4 完成后 | AK/SK 认证流程是否顺畅？签名算法两端一致？ | 若不一致，PA-3 同步修复 |
| **CK-2** | PA-5 完成后 | `@opencode-ai/sdk` event.subscribe() 是否稳定？事件格式是否符合预期？ | 若不稳定，评估 SDK 版本或降级方案 |
| **CK-3** | GW-7 + SK-4 完成后 | AI-Gateway ↔ Skill Server WebSocket 连接是否稳定？ | 增加心跳检测和重连逻辑 |
| **CK-4** | SC-8 完成后 | 流式渲染性能是否达标？大段代码是否卡顿？ | 引入消息分块渲染或 Web Worker |
| **CK-5** | SC-15 前 | 端到端链路延迟是否 ≤ 100ms？ | 定位瓶颈节点，优化序列化/网络 |
| **CK-6** | SC-14 完成后 | Skill 小程序与 IM 聊天是否有样式/布局冲突？ | 样式隔离（CSS Modules / Shadow DOM） |
| **CK-7** | Phase 1 整体 | IM 聊天功能是否有回归？ | 回归测试清单（发消息/收消息/群聊/文件/表情） |

### 6.2 Phase 2 风险检查点

| 检查点 | 时间节点 | 检查内容 | 应对措施 |
|---|---|---|---|
| **CK-8** | SK-10 完成后 | IM 平台 API 调用是否有权限限制？频率限制？ | 提前申请权限；加频率限制保护 |
| **CK-9** | GW-10 完成后 | Redis Pub/Sub 跨实例中继是否可靠？延迟如何？ | 增加消息确认机制；监控中继延迟 |
| **CK-10** | SC-20 完成后 | 1000+ 消息场景下虚拟滚动是否流畅？ | 压测 + 优化渲染批次 |

## 7. 风险总览与应对

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| OpenCode SDK API 变更 | 中 | 高 | `@opencode-ai/sdk` 版本锁定 `^1.2`；PCAgent 按 semver 升级 |
| PCAgent WebSocket 断连 | 高 | 中 | 指数退避重连（1s→2s→4s，上限 30s）；心跳超时推 `agent_offline` |
| AI-Gateway ↔ Skill Server 连接断开 | 中 | 中 | 指数退避重连；断开期间缓冲到 Redis |
| Skill 小程序性能（大量消息） | 中 | 中 | 虚拟滚动（react-window）；消息分页加载（每次 50 条） |
| IM 消息发送权限 | 低 | 中 | AK/SK 权限校验；仅允许发送到用户自己的聊天 |
| OpenCode 本地未安装 | 高 | 低 | PCAgent 启动时检测可达性；不可达时 UI 显示安装引导 |
| 多 PCAgent 实例冲突 | 中 | 中 | 同一 AK + 同一工具类型只允许一个活跃连接；新连接踢掉旧连接 |
| 两个独立服务部署协调 | 中 | 中 | API 版本化；内部通信协议向后兼容；灰度发布策略 |
| 小程序容器兼容性 | 中 | 高 | 提前与 IM 平台确认小程序 API 可用性；预留 fallback 方案 |
| AK/SK 签名算法两端不一致 | 中 | 高 | CK-1 检查点；签名算法文档化 + 联调测试 |
