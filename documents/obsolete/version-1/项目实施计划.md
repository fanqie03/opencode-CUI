# OpenCode-CUI 项目实施计划

**版本**: v2.0 | **日期**: 2026-03-04 | **状态**: 已确认

---

## 一、项目概述

### 1.1 项目目标

分两阶段将本地 AI 编码工具（OpenCode）集成到用户界面：

- **Demo 阶段 (8 周)**：开发本地联调 Demo，验证 OpenCode → PCAgent → AI-Gateway → Skill Server → Web UI 端到端技术链路
- **最终实现 (12 周)**：完成 IM 平台集成、多工具支持、团队协作、生产级部署

### 1.2 项目范围

| 维度 | Demo 阶段 | 最终实现阶段 |
|------|-----------|-------------|
| 客户端 | React Web UI | IM Skill 小程序 |
| PCAgent | 独立 Node.js 进程 | IM Electron 插件 |
| AI 工具 | 仅 OpenCode | OpenCode + Claude Code + OpenClaw |
| 协作 | 个人私有 | 团队共享 |
| 部署 | 本地 + 简单服务端 | 高可用集群 |
| 用户规模 | 10 人内测 | 10000+ |

### 1.3 关键假设和约束

- OpenCode 使用默认配置（localhost:54321），无特殊要求
- 数据库使用 MySQL 5.7
- PCAgent 代码已存在于 `src/main/pc-agent/`
- AI-Gateway 和 Skill Server 后端代码已有基础实现
- IM 平台集成需要与 IM 团队协调接口规范

## 五、风险管理

### 5.1 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| OpenCode SDK API 变更 | 中 | 高 | 版本锁定 + semver 升级策略 |
| PCAgent 内存泄漏 | 中 | 中 | 24h 压测 + 健康检查 + 自动重启 |
| Redis Pub/Sub 消息丢失 | 中 | 中 | 序列号检测 + 数据库备份 |
| WebSocket 长连接不稳定 | 低 | 中 | 心跳保活 + 指数退避重连 |
| 前端渲染性能 | 低 | 低 | 虚拟滚动 + 懒加载 + React.memo |
| MySQL 连接池耗尽 | 低 | 高 | 监控告警 + 连接池调优 + 限流 |

### 5.2 进度风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| Web UI 开发时间不足 | 中 | 中 | 优先核心功能，UI 美化延后 |
| 多实例协调复杂度高 | 中 | 低 | 先单实例验证，再扩展多实例 |
| IM 平台技术对接延迟 | 中 | 高 | Demo 先行，降低对 IM 的依赖 |
| 安全审计耗时超预期 | 低 | 中 | 提前进行安全设计和自查 |

### 5.3 外部依赖风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| IM 平台接口不稳定 | 中 | 高 | 接口 Mock + 降级方案 |
| OpenCode 版本不兼容 | 中 | 中 | 协议版本检测 + 兼容层 |
| IM 团队配合不到位 | 低 | 高 | 提前签署对接文档 + 定期同步 |
