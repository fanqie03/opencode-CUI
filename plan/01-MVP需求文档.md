# opencode-CUI — MVP 需求文档

> 版本：V2 | 日期：2026-03-02 | 关联：`00-方案设计主文档.md`

---

## 1. 需求背景

### 1.1 现状与痛点

现有聊天工具（类 DingTalk/Feishu/Slack）的 IM 消息仅支持纯文本，无法承载 AI 编码工具的富文本输出：

| 痛点 | 描述 |
|---|---|
| 无代码高亮 | AI 返回的代码块在 IM 中以纯文本显示，可读性差 |
| 无流式响应 | AI 生成内容需等待完成后一次性展示，体验延迟大 |
| 无 Markdown 渲染 | 标题、列表、表格等格式化内容无法正确展示 |
| 无 Tool Use 展示 | AI 调用工具（文件读写、终端命令等）的过程和结果无法可视化 |
| 上下文割裂 | 开发者需在 IM 和 AI 工具之间频繁切换，工作流被打断 |

### 1.2 解决方案

通过 IM 已有的 "/" 命令框架触发 **Skill 交互**——用户选择 SKILL 后，IM 聊天框上方出现微型状态条（Mini Bar），可展开为 Skill 小程序客户端，在其中与 AI 工具（OpenCode）进行多轮对话。对话结果可选中发送回 IM 聊天。

### 1.3 MVP 范围

| 维度 | MVP 范围 | 后续迭代（不在本次范围） |
|---|---|---|
| AI 工具 | **仅 OpenCode** 本地模式 | Claude Code、OpenClaw、远程模式 |
| 共享模式 | **个人私有**会话 | 团队共享 |
| 试点平台 | **Windows PC**（Electron + React + TS） | Mac、Linux、Web |
| 部署模式 | AI 工具与客户端同机（localhost） | 远程 AI 工具实例 |

## 2. 目标用户

### 2.1 用户画像

- **角色**：使用现有聊天工具（类企业 IM）的软件开发者
- **环境**：Windows PC，已安装聊天客户端（Electron），本地已安装 OpenCode
- **日常场景**：在 IM 中沟通的同时需要使用 AI 编码辅助工具
- **核心诉求**：不离开 IM 即可与 AI 编码工具交互

### 2.2 使用前提

- 用户已在本地安装并启动 OpenCode（`opencode serve`，默认 `localhost:54321`）
- 用户已获取聊天平台开放平台的 AK/SK 并在客户端中配置
- 用户所在组织已部署 AI-Gateway 和 Skill Server

## 3. 用户故事

### US-1：触发 Skill

> **作为**一名开发者，
> **我希望**在 IM 输入框输入 "/" 后直接看到 OpenCode SKILL 选项并点击触发，
> **以便**我无需离开 IM 就能启动 AI 编码对话。

**验收标准**：
- 输入 "/" 后 SKILL 列表中显示 "OpenCode" 选项（含图标和描述）
- 点击后 IM 聊天框出现一行高亮 SKILL 名称（如 "✦ OpenCode"）
- 用户可在 IM 输入框中继续输入问题

**边界条件与异常场景**：
- 网络断开时：SKILL 列表仍可显示（本地缓存），点击后提示"网络连接失败"
- OpenCode 未启动：SKILL 仍可选择，但点击"生成"后显示"工具离线"
- 用户快速连续点击 SKILL：仅触发一次，后续点击被忽略（防重复）
- 输入框已有内容时选择 SKILL：SKILL 行插入到输入框前，用户输入保留
- 用户取消 SKILL 选择：点击其他命令或按 Esc 键，SKILL 行消失，输入框恢复

### US-2：发送问题并查看 Mini Bar
> **作为**一名开发者，
> **我希望**输入问题点击"生成"后看到处理状态，
> **以便**我知道 AI 正在处理我的请求。

**验收标准**：
- 点击"生成"后 IM 聊天区出现 1 行高的 Mini Bar
- Mini Bar 显示 "OpenCode 处理中…" 状态
- Mini Bar 右侧有"展开"按钮

**边界条件与异常场景**：
- 用户输入为空：点击"生成"时禁用按钮，提示"请输入问题"
- 网络超时（>30s）：Mini Bar 显示"请求超时，请重试"，提供重试按钮
- OpenCode 崩溃：Mini Bar 显示"工具异常，请检查本地 OpenCode"
- 用户快速连续点击"生成"：仅发送第一次请求，后续点击被忽略直到响应完成
- Mini Bar 显示期间用户关闭 IM：重新打开 IM 后 Mini Bar 状态恢复
- 消息过长（>5000 字符）：截断显示，提示"消息已截断"

### US-3：展开 Skill 小程序进行多轮对话

> **作为**一名开发者，
> **我希望**点击"展开"后能在 Skill 小程序中查看完整 AI 响应并继续对话，
> **以便**我能进行深度的多轮 AI 编码交互。

**验收标准**：
- 点击"展开"后 Skill 小程序覆盖 IM 聊天窗上方
- AI 响应以流式方式逐字显示
- 代码块有语法高亮和"复制"按钮
- 可在输入框中继续输入问题，支持多轮对话
- 支持 Shift+Enter 换行

**边界条件与异常场景**：
- 展开期间网络断开：显示"连接已断开"，提供重新连接按钮
- AI 响应超长（>10000 字符）：分页显示，底部提供"加载更多"按钮
- 用户在展开状态下关闭 IM：重新打开后 Skill 小程序自动恢复展开状态
- 代码块超大（>500 行）：显示行号，提供"全屏查看"按钮
- 用户快速切换会话：前一个会话的流式消息停止，新会话消息开始推送
- 输入框输入超长文本（>2000 字符）：提示"输入过长"，限制输入
- 用户在流式渲染中点击收起：流式渲染停止，消息保存到数据库

### US-4：选文发送到 IM

> **作为**一名开发者，
> **我希望**选中 AI 回复中的任意文本并发送到 IM 聊天，
> **以便**我能将 AI 生成的代码或说明分享给同事。

**验收标准**：
- 选中 AI 回复文本时浮现"发送到聊天"按钮
- 点击后文本作为 IM 消息发送到当前聊天
- 发送成功后有提示反馈

**边界条件与异常场景**：
- 选中空文本：不显示"发送到聊天"按钮
- 选中代码块：保留代码格式（Markdown 或纯文本），发送到 IM
- 选中包含特殊字符的文本：正确转义，IM 中显示正确
- 发送到 IM 失败（权限不足）：显示"发送失败：权限不足"
- 用户快速连续点击"发送到聊天"：仅发送一次，后续点击被忽略
- 选中文本后网络断开：按钮禁用，提示"网络连接失败"
- 发送超大文本（>10000 字符）：分段发送或提示"文本过长，请分段发送"

### US-5：会话历史恢复

> **作为**一名开发者，
> **我希望**重新打开 Skill 小程序后能看到之前的对话记录，
> **以便**我能继续之前的对话上下文。

**验收标准**：
- 关闭并重新展开 Skill 小程序后，历史消息从数据库恢复
- 消息顺序正确，代码块仍有语法高亮
- 可查看和切换多个历史会话

**边界条件与异常场景**：
- 会话消息超过 1000 条：分页加载，首次加载最近 50 条，向上滚动加载更多
- 数据库连接失败：显示"无法加载历史消息"，提供重试按钮
- 消息包含已删除的用户信息：显示"[已删除用户]"
- 会话创建后未发送任何消息：重新打开时显示空会话
- 用户在加载历史消息期间切换会话：停止当前加载，开始加载新会话
- 历史消息中包含已过期的代码块：仍正常显示，但提示"此代码可能已过期"
- 会话被管理员删除：显示"会话已被删除"，提供返回列表按钮

### US-6：离线状态提示

> **作为**一名开发者，
> **我希望**在 OpenCode 未启动时能得到明确提示，
> **以便**我知道需要先启动本地 AI 工具。

**验收标准**：
- 本地 OpenCode 未启动时，Mini Bar 显示 "工具离线"
- Skill 小程序内显示离线状态指示
- OpenCode 重新启动后自动恢复连接，状态更新为在线

**边界条件与异常场景**：
- OpenCode 启动中（未完全就绪）：显示"工具启动中…"，禁用发送按钮
- OpenCode 进程崩溃：Mini Bar 自动更新为"工具异常"，提供"重启工具"按钮
- 网络连接丢失但 OpenCode 仍运行：显示"网络连接失败"（区别于工具离线）
- 用户在离线状态下输入问题：点击"生成"时提示"工具离线，请先启动 OpenCode"
- PCAgent 与 OpenCode 连接超时（>30s）：自动标记为离线
- 用户快速启动/关闭 OpenCode：状态变化频繁时，UI 显示最终状态
- 离线期间用户关闭 IM：重新打开后仍显示离线状态，直到 OpenCode 恢复

### US-7：无回归保证

> **作为**一名普通 IM 用户，
> **我希望**IM 的聊天功能不受 Skill 集成影响，
> **以便**我的日常沟通不被干扰。

**验收标准**：
- 不使用 Skill 时，IM 聊天功能完全正常
- Skill 小程序收起后，IM 聊天窗恢复原状
- PCAgent 插件不影响 IM 客户端启动速度和稳定性

**边界条件与异常场景**：
- PCAgent 启动失败：IM 客户端仍正常启动，Skill 不可用，显示"Skill 加载失败"
- PCAgent 占用过多内存（>200MB）：监控告警，但不影响 IM 功能
- 用户快速切换 "/" 命令：其他命令仍正常响应，无延迟
- Skill 小程序崩溃：IM 聊天功能不受影响，用户可重新打开 Skill
- IM 客户端启动时 PCAgent 初始化中：IM 启动不阻塞，PCAgent 后台初始化
- 用户禁用 Skill：IM 聊天功能完全不受影响
- 多个 "/" 命令同时触发：各命令独立运行，互不干扰

## 5. 非功能需求

| 编号 | 维度 | 要求 | 验证方法 |
|---|---|---|---|
| NFR-1 | **性能 - 首字节延迟** | AI 首字节到达后 ≤200ms 内开始渲染 | Chrome DevTools Performance 面板，记录 AI 响应首字节时间戳与 UI 首次渲染时间戳 |
| NFR-2 | **性能 - 消息列表** | 消息列表支持 1000+ 消息不卡顿（虚拟滚动），滚动帧率 ≥55fps | 加载 1000 条消息，上下滚动，测量帧率 |
| NFR-3 | **性能 - 流式渲染** | 流式消息渲染无闪烁，每个 delta 事件处理 <10ms | 监控 delta 事件处理时间 |
| NFR-4 | **可用性 - 多实例** | AI-Gateway 和 Skill Server 支持多实例部署，单实例故障不影响整体服务 | 部署 2 个实例，关闭其中一个，验证服务继续可用 |
| NFR-5 | **延迟 - 端到端中继** | 端到端消息中继延迟（PCAgent → Skill 小程序）≤100ms（不含 AI 生成时间） | 在 PCAgent 和 Skill 小程序 WebSocket 层记录时间戳，计算差值 |
| NFR-6 | **延迟 - 会话创建** | 创建新会话 ≤500ms | 从点击"新建会话"到会话列表更新的时间 |
| NFR-7 | **安全 - 认证** | AK/SK 签名认证防重放（5min 时间窗口）；WebSocket 传输使用 WSS | 测试过期签名被拒绝；验证 WSS 连接 |
| NFR-8 | **安全 - 数据隔离** | 用户只能访问自己的会话和消息 | 尝试访问其他用户的会话，验证被拒绝 |
| NFR-9 | **兼容性 - 回归** | 不影响现有 IM 聊天功能（零回归） | 执行 IM 功能回归测试清单（见第 7 节） |
| NFR-10 | **兼容性 - 启动速度** | PCAgent 不影响客户端启动速度（增量 ≤500ms） | 对比加载 PCAgent 前后的启动时间 |
| NFR-11 | **可维护性 - 协议适配** | OpenCode 协议适配集中在 Skill 小程序 `protocol/` 模块，服务端透传不解析 | 代码审查，验证 AI-Gateway 和 Skill Server 无 OpenCode 特定逻辑 |
| NFR-12 | **数据持久化** | 所有会话和消息持久化到 MySQL，支持历史查询；消息保存延迟 ≤1s | 验证消息在数据库中可查询；测量消息保存延迟 |
| NFR-13 | **可观测性 - 日志** | 关键链路日志（连接建立/断开、消息收发、错误）；日志级别可配置 | 检查日志输出，验证关键事件被记录 |
| NFR-14 | **可观测性 - 健康检查** | 健康检查端点 `/health`，返回各组件状态 | 调用健康检查端点，验证返回正确状态 |
| NFR-15 | **可观测性 - 监控** | 关键指标：连接数、消息吞吐量、错误率、延迟分布 | 部署 Prometheus 采集指标 |

## 8. 范围边界

### 8.1 本次范围内（In Scope）

- 4 个新建组件：PCAgent Plugin、AI-Gateway、Skill Server、Skill 小程序
- OpenCode 本地模式（localhost:54321）通过 `@opencode-ai/sdk` 集成
- AK/SK 认证机制
- "/" 命令触发 → Mini Bar → Skill 小程序 → 多轮对话 → 选文发送 IM
- 会话和消息持久化（MySQL）
- 流式渲染（Markdown + 代码高亮 + Tool Use）
- PCAgent 断线重连和健康检测
- AI-Gateway 和 Skill Server 多实例 HA 基础支持

| 团队共享会话 | 多人共享同一 AI 对话 |
| Mac/Linux 客户端 | 首期仅 Windows PC |

| 语音/图片输入 | 多模态输入 |
| 权限管理后台 | AK/SK 管理 UI |
| 监控告警 | Prometheus/Grafana 集成 |

## 9. MVP 优先级与延后项

### 9.1 Phase 1 必须完成（P0 功能）

这些功能是 MVP 的核心，必须在 Phase 1 完成：

| 功能 | 理由 | 验收标准 |
|---|---|---|
| "/" 命令触发 Skill | 用户入口，无此功能无法使用 | AC-1 |
| Mini Bar 状态显示 | 用户反馈，必须有处理状态提示 | AC-2 |
| Skill 小程序展开 | 核心交互，必须支持多轮对话 | AC-3 |
| 流式渲染 + 代码高亮 | 核心价值，代码可读性 | AC-4 |
| AK/SK 认证 + 长连接 | 安全性和可靠性基础 | AC-6 |
| 历史消息恢复 | 用户体验，不能丢失对话 | AC-7 |
| 离线状态提示 | 用户体验，必须明确告知工具状态 | AC-8 |
| IM 功能不受影响 | 零回归，不能破坏现有功能 | AC-9 |

### 9.2 Phase 2 可选项（P1 功能）

这些功能增强用户体验，可在 Phase 2 实现：

| 功能 | 理由 | 优先级 |
|---|---|---|
| 选文发送到 IM | 便利性，但不是必须 | P1 |
| 会话列表 Sidebar | 便利性，但可用单会话替代 | P1 |
| 消息分页加载 | 性能优化，初期消息量不大 | P1 |
| Tool Use 可视化 | 增强功能，但不影响基本使用 | P1 |
| 多实例 HA（Redis） | 可靠性，初期可单实例部署 | P1 |

### 9.3 后续迭代（不在 MVP 范围）

| 功能 | 理由 | 预计迭代 |
|---|---|---|
| Claude Code 集成 | 需要 subprocess bridge，复杂度高 | v1.1 |
| OpenClaw 集成 | 需要 WebSocket RPC bridge，复杂度高 | v1.1 |
| 远程模式 | 需要网络隔离和权限管理 | v1.2 |
| 团队共享会话 | 需要权限系统和冲突解决 | v1.2 |

## 10. 关键风险与应对

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| OpenCode SDK API 变更 | 中 | 高 | 版本锁定 `@opencode-ai/sdk@^1.2`；定期检查更新 |
| PCAgent WebSocket 断连 | 高 | 中 | 指数退避重连（1s→2s→4s，上限 30s）；心跳检测 |
| AI-Gateway ↔ Skill Server 连接断开 | 中 | 中 | 双向重连；Redis 缓冲（Phase 2） |
| Skill 小程序性能（大消息） | 中 | 中 | 虚拟滚动；消息分页加载 |
| IM 消息发送权限不足 | 低 | 高 | AK/SK 权限校验；仅允许发送到用户自己的聊天 |
| OpenCode 本地未安装 | 高 | 高 | PCAgent 启动时检测；UI 显示安装引导 |
| 多 PCAgent 实例冲突 | 中 | 中 | 同一 AK + 工具类型只允许一个活跃连接；新连接踢掉旧连接 |
| 两个独立服务部署协调 | 中 | 中 | API 版本化；内部通信协议向后兼容；灰度发布 |
