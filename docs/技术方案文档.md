## 2. 关键技术实现细节

### 2.1 透传模式架构

**核心设计原则**：服务端不解析 OpenCode 事件，保持协议独立性。

```
OpenCode 事件
    ↓ (原始 JSON)
PCAgent event.subscribe()
    ↓ (WebSocket 透传)
AI-Gateway EventRelayService
    ↓ (WebSocket 透传)
Skill Server GatewayWSHandler
    ↓ (WebSocket 推流)
Skill 小程序 protocol/ 解析
    ↓ (类型化事件)
UI 渲染
```

**优势**：
- 服务端无需关心 OpenCode 版本更新
- 协议适配集中在客户端，易于维护
- 支持未来多工具扩展

### 2.2 双服务架构职责分离

**AI-Gateway（端口 8081）**：
- 连接管理：PCAgent 注册、心跳、上下线
- 认证层：AK/SK 签名验证
- 中继层：双向消息透传（不解析）
- 无状态设计：支持水平扩展

**Skill Server（端口 8082）**：
- 会话管理：创建、查询、关闭
- 消息持久化：存储原始 OpenCode 事件
- 推流服务：WebSocket 推送给小程序
- IM 集成：调用 IM 平台 API 发送消息

### 2.3 Redis Pub/Sub 多实例协调

**Agent 连接注册**：
```
Key:  gw:agent:conn:{agentId}
Val:  {"instanceId":"gw-node-1","userId":123,"toolType":"OPENCODE"}
TTL:  90s（心跳每 30s 刷新）
```

**跨实例消息中继**：
```
Channel: gw:relay:{targetInstanceId}
Message: GatewayMessage JSON
```

**路由逻辑**：
```java
if (agentOnLocalInstance) {
    localAgentPool.send(agentId, msg);
} else {
    redis.publish("gw:relay:" + targetInstance, msg);
}
```

### 2.4 AK/SK 认证机制

**签名算法**：
```
signature = HMAC-SHA256(
  key = SK,
  message = "{AK}\n{timestamp}\n{nonce}"
)
```

**验证流程**：
1. 检查 timestamp 窗口（±5 分钟）
2. 检查 nonce 防重放（Redis SET NX，TTL 5 分钟）
3. 查询 AK 对应的 SK
4. 计算期望签名，常时间比对
5. 验证通过建立 WebSocket 连接

**防护措施**：
- 时间戳防重放：5 分钟窗口
- Nonce 防重复：Redis 存储已用 nonce
- 常时间比对：防止时序攻击

### 2.5 会话生命周期管理

**状态转移**：
```
ACTIVE → IDLE → CLOSED
  ↑       ↓
  └─ touchSession()
```

**自动清理**：
- 定时任务每 10 分钟执行一次
- 将 30 分钟未活动的 ACTIVE 会话标记为 IDLE
- 支持手动关闭会话

**消息序列化**：
- 每条消息包含 seq 字段
- Redis Pub/Sub 自动注入 sequenceNumber
- 防止消息乱序

## 3. 接口设计规范

### 3.1 REST API 规范

**基础 URL**：
- AI-Gateway: `http://localhost:8081/api/gateway`
- Skill Server: `http://localhost:8082/api/skill`

**响应格式**：
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

**错误码**：
- 0: 成功
- 400: 请求参数错误
- 401: 认证失败
- 403: 权限不足
- 404: 资源不存在
- 500: 服务器错误

### 3.2 Skill Server REST API

**会话管理**：
```
POST   /api/skill/sessions              创建会话
GET    /api/skill/sessions              查询会话列表（分页）
GET    /api/skill/sessions/{id}         获取会话详情
DELETE /api/skill/sessions/{id}         关闭会话
```

**消息管理**：
```
POST   /api/skill/sessions/{id}/messages        发送用户消息
GET    /api/skill/sessions/{id}/messages        获取消息历史
POST   /api/skill/sessions/{id}/send-to-im      发送到 IM
```

**SKILL 定义**：
```
GET    /api/skill/definitions           查询可用 SKILL 列表
```

### 3.3 WebSocket 消息格式

**PCAgent → AI-Gateway**：
```json
{"type":"register","deviceName":"...","os":"WINDOWS","toolType":"OPENCODE","toolVersion":"1.2.15"}
{"type":"heartbeat"}
{"type":"tool_event","sessionId":"...","event":{...}}
{"type":"tool_done","sessionId":"...","usage":{"input_tokens":100,"output_tokens":200}}
{"type":"tool_error","sessionId":"...","error":"..."}
{"type":"session_created","toolSessionId":"..."}
```

**AI-Gateway → Skill Server**：
```json
{"type":"tool_event","agentId":"...","sessionId":"...","event":{...}}
{"type":"tool_done","agentId":"...","sessionId":"...","usage":{...}}
{"type":"agent_online","agentId":"...","toolType":"OPENCODE","toolVersion":"1.2.15"}
{"type":"agent_offline","agentId":"..."}
```

**Skill Server → Skill 小程序**：
```json
{"type":"delta","seq":1,"content":"..."}
{"type":"done","seq":2,"usage":{"input_tokens":100,"output_tokens":200}}
{"type":"error","message":"..."}
{"type":"agent_offline"}
{"type":"agent_online"}
```

## 4. 数据流详解

### 4.1 消息发送流（用户 → AI）

```
1. 用户在 Skill 小程序输入问题
   ↓
2. Skill 小程序 POST /api/skill/sessions/{id}/messages
   {
     "content": "请帮我写一个函数",
     "role": "USER"
   }
   ↓
3. Skill Server 持久化消息到 MySQL
   ↓
4. Skill Server 通过 WebSocket 发送到 AI-Gateway
   {
     "type": "invoke",
     "agentId": "agent_001",
     "sessionId": "sess_123",
     "action": "chat",
     "payload": {
       "text": "请帮我写一个函数",
       "toolSessionId": "oc_sess_456"
     }
   }
   ↓
5. AI-Gateway 通过 WebSocket 发送到 PCAgent
   ↓
6. PCAgent 调用 opencode.session.chat()
   ↓
7. OpenCode 处理请求
```

### 4.2 消息接收流（AI → 用户）

```
1. OpenCode 生成 SSE 事件流
   {
     "type": "message.updated",
     "sessionId": "oc_sess_456",
     "delta": "这是一个示例函数"
   }
   ↓
2. PCAgent event.subscribe() 接收事件
   ↓
3. PCAgent 通过 WebSocket 透传到 AI-Gateway
   {
     "type": "tool_event",
     "sessionId": "sess_123",
     "event": { /* OpenCode 原始事件 */ }
   }
   ↓
4. AI-Gateway 通过 WebSocket 透传到 Skill Server
   ↓
5. Skill Server 持久化消息到 MySQL
   ↓
6. Skill Server 通过 WebSocket 推流到 Skill 小程序
   {
     "type": "delta",
     "seq": 1,
     "content": "这是一个示例函数"
   }
   ↓
7. Skill 小程序 protocol/ 模块解析事件
   ↓
8. UI 渲染消息
```

### 4.3 选文发送到 IM 流

```
1. 用户在 Skill 小程序选中 AI 回复文本
   ↓
2. 浮现"发送到聊天"按钮
   ↓
3. 用户点击按钮
   ↓
4. Skill 小程序 POST /api/skill/sessions/{id}/send-to-im
   {
     "content": "选中的文本",
     "chatId": "im_chat_123"
   }
   ↓
5. Skill Server 调用 IM 平台 API
   POST {IM_API_URL}/messages
   {
     "chatId": "im_chat_123",
     "content": "选中的文本",
     "source": "skill"
   }
   ↓
6. IM 聊天窗显示消息
```

### 4.4 时序图

```
Skill小程序   Skill Server   AI-Gateway   PCAgent    OpenCode
     │              │       │         │        │
     │─ POST msg ──►│              │           │           │
     │              │─ save msg ───│           │           │
     │            │─ WS send ───►│           │        │
     │          │              │─ WS send ►│         │
     │              │              │      │─ chat() ─►│
     │         │              │           │         │
     │            │              │           │◄─ SSE ────│
     │              │              │◄─ WS ─────│           │
     │              │◄─ WS ────────│           │         │
     │            │─ save msg ───│           │         │
     │◄─ WS stream ─│       │           │           │
     │              │              │     │
  [选文→发送IM]     │      │           │           │
     │─ POST toIM ─►│            │           │
     │              │─ call IM API ─────────── IM Server  │
```

## 5. 性能优化方案

### 5.1 连接管理优化

**连接池配置**：
```yaml
# AI-Gateway
datasource:
  hikari:
    maximum-pool-size: 20
    minimum-idle: 5
    idle-timeout: 300000
    connection-timeout: 20000

# Redis
lettuce:
  pool:
    max-active: 50
    max-idle: 20
    min-idle: 5
```

**WebSocket 连接管理**：
- 心跳间隔：30 秒
- 心跳超时：90 秒
- 断线重连：指数退避（1s → 2s → 4s → 8s → 16s → 30s）

### 5.2 消息流优化

**流式处理**：
- OpenCode 事件通过 SSE 流式传输
- Skill 小程序 delta 事件实时渲染
- 避免等待完整消息，提升用户体验

**消息缓冲**：
- Redis Pub/Sub 自动缓冲消息
- 多实例间消息不丢失
- 支持消费者离线重连

### 5.3 数据库优化

**索引策略**：
```sql
-- AI-Gateway
CREATE INDEX idx_user ON agent_connection(user_id);
CREATE INDEX idx_ak ON agent_connection(ak_id);
CREATE INDEX idx_status ON agent_connection(status);

-- Skill Server
CREATE INDEX idx_user_active ON skill_session(user_id, status);
CREATE INDEX idx_agent ON skill_session(agent_id);
CREATE INDEX idx_session_seq ON skill_message(session_id, seq);
```

**查询优化**：
- 使用分页查询，避免全表扫描
- 会话列表按 status 过滤
- 消息历史按 seq 排序

### 5.4 缓存策略

**Redis 缓存**：
- Agent 连接注册（TTL 90s）
- Nonce 防重放（TTL 5 分钟）
- 会话推流订阅关系

**缓存失效**：
- Agent 下线时清除连接记录
- 会话关闭时清除推流订阅
- 定时清理过期数据

## 6. 安全机制

### 6.1 认证安全

**AK/SK 认证**：
- HMAC-SHA256 签名算法
- 时间戳防重放（±5 分钟）
- Nonce 防重复（Redis 存储）
- 常时间比对防时序攻击

**内部服务认证**：
- AI-Gateway ↔ Skill Server 使用 internal token
- Token 通过环境变量配置
- WebSocket 握手时验证

### 6.2 传输安全

**生产环境**：
- 使用 WSS（WebSocket Secure）
- 启用 TLS 1.2+
- 证书定期更新

**开发环境**：
- 允许 WS（WebSocket）
- 本地测试无需 TLS

### 6.3 数据安全

**敏感信息处理**：
- SK 不存储在代码中，使用环境变量
- 数据库密码加密存储
- 日志不输出敏感信息

**SQL 注入防护**：
- 使用 MyBatis 参数化查询
- 避免字符串拼接 SQL

### 6.4 访问控制

**用户隔离**：
- 每个会话绑定 userId
- 查询时过滤 userId
- 防止跨用户数据访问

**Agent 隔离**：
- 每个 Agent 绑定 AK
- 同一 AK 同一工具类型只允许一个活跃连接
- 新连接踢掉旧连接

## 8. 故障恢复策略

### 8.1 常见故障场景

| 故障场景 | 症状 | 恢复策略 |
|---|---|---|
| PCAgent 断连 | 用户无法发送消息 | 指数退避重连；心跳超时推 agent_offline |
| AI-Gateway 单节点宕机 | 部分用户连接断开 | LB 自动切换；PCAgent 重连到其他节点 |
| Skill Server 单节点宕机 | 推流中断 | LB 自动切换；无状态可即时恢复 |
| Gateway ↔ Skill Server 连接断开 | 消息无法中继 | Skill Server 指数退避重连；缓冲到 Redis |
| Redis 不可用 | 多实例协调失败 | 降级为本地路由（单实例模式） |
| MySQL 不可用 | 消息持久化失败 | 返回错误；推流不受影响 |

### 8.2 恢复流程

**PCAgent 断线重连**：
```
连接断开
  ↓
等待 1s
  ↓
尝试重连
  ↓
成功 → 重置延迟
失败 → 延迟翻倍（上限 30s）
```

**AI-Gateway ↔ Skill Server 重连**：
```
连接断开
  ↓
Skill Server 检测到断开
  ↓
等待 1s
  ↓
尝试重连
  ↓
成功 → 重置延迟
失败 → 延迟翻倍（上限 30s）
```

### 8.3 数据一致性保证

**消息序列化**：
- 每条消息包含 seq 字段
- 接收端按 seq 排序
- 防止消息乱序

**会话状态同步**：
- 会话状态存储在 MySQL
- 多实例通过数据库查询获取最新状态
- 避免状态不一致

## 9. 扩展性设计

### 9.1 水平扩展

**AI-Gateway 扩展**：
- 无状态设计，可直接增加实例
- Load Balancer 自动分发连接
- Redis 存储 Agent 连接注册

**Skill Server 扩展**：
- 无状态设计，可直接增加实例
- 会话数据存储在 MySQL
- 推流状态存储在 Redis

### 9.2 协议兼容性

**当前实现**：
- PCAgent 通过 @opencode-ai/sdk 订阅 OpenCode SSE 事件
- 全链路透传，不做协议转换
- Skill 小程序 protocol/ 模块负责解析

**版本兼容**：
- OpenCode 更新事件格式 → 仅需更新 Skill 小程序 protocol/ 模块
- AI-Gateway、Skill Server、PCAgent 无需改动

**多工具扩展**（未来）：
- Skill 小程序新增对应 protocol/XxxParser.ts
- PCAgent 新增对应 Bridge 模块
- 服务端保持透传，无需改动

### 9.3 功能扩展点

**会话功能**：
- 支持会话分享
- 支持会话导出
- 支持会话搜索

**消息功能**：
- 支持消息编辑
- 支持消息删除

**IM 集成**：
- 支持多个 IM 平台
