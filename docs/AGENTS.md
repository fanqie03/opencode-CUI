# 技术文档

**Last Updated:** 2026-03-04

## 目的

存放技术文档、协议规范、架构分析等深度技术资料。本目录是项目技术实现的参考源，所有架构设计、协议规范和技术方案均源于此。

## 关键文件

| 文件 | 描述 |
|------|------|
| `技术方案文档.md` | 详细技术方案，包括技术选型、实现细节、接口设计 |
| `架构设计详解.md` | 深度架构分析，组件详解，设计模式 |
| `PROTOCOL_SPEC.md` | 消息信封协议规范，序列号机制 |

## AI Agent 工作指南

### 在此目录工作时

**修改技术方案文档：**
- 保持原有结构和格式（Markdown 表格、代码块、列表）
- 技术选型变更时更新对应的理由说明
- 新增技术方案时补充实现细节和代码示例
- 所有代码示例应基于实际实现，可运行验证
- 更新时同步修改相关的架构设计详解

**架构变更：**
- 架构变更需同步更新 `架构设计详解.md`
- 更新组件职责、通信协议、数据流图
- 检查是否影响其他文档（技术方案、协议规范）
- 在 `技术方案文档.md` 中记录技术选型变更

**协议规范更新：**
- 协议变更需更新 `PROTOCOL_SPEC.md`
- 包括消息结构、字段定义、示例
- 记录版本号和向后兼容性说明
- 更新实现指南（TypeScript、Java 代码示例）

**添加新技术文档：**
- 在本目录创建新的 `.md` 文件
- 更新本 AGENTS.md 中的"关键文件"表格
- 文档应包含：概述、设计原理、实现细节、代码示例
- 与相关文档建立交叉引用

### 文档维护规范

**格式要求：**
- 所有文档使用 Markdown 格式
- 表格用于结构化信息（技术选型、组件对比、字段定义）
- 代码块用于示例和配置（JSON、SQL、TypeScript、Java）
- 列表用于步骤、要点、检查清单
- 标题层级：# 主标题，## 二级标题，### 三级标题

**代码示例验证：**
- 所有代码示例必须基于实际代码库
- TypeScript 示例应来自 `src/main/pc-agent/` 或 `skill-miniapp/src/`
- Java 示例应来自 `ai-gateway/src/` 或 `skill-server/src/`
- 示例应包含必要的导入和上下文
- 复杂示例应附带说明注释

**架构图：**
- 使用 ASCII 或 Mermaid 格式
- 清晰标注组件、通信协议、数据流
- 包含端口号、服务名称、关键路径
- 保持图表与实际架构同步

**交叉引用：**
- 技术方案中的技术选型应对应架构文档中的组件
- 协议规范应被 PC Agent 和 Skill Server 实现引用
- 使用相对路径引用：`[架构设计详解](./架构设计详解.md)`
- 引用 plan/ 目录文档时使用：`[需求文档](../plan/01-MVP需求文档.md)`

**版本控制：**
- 每个文档顶部包含"Last Updated"时间戳
- 重大变更在文档中添加变更日志（Changelog）
- 保持 Git 提交信息清晰（关联文档变更）
- 协议规范需记录版本号和兼容性说明

### 常见任务

**添加新技术方案：**
1. 在 `技术方案文档.md` 中添加技术选型和理由
2. 在 `架构设计详解.md` 中描述组件实现
3. 添加代码示例（TypeScript 或 Java）
4. 更新本 AGENTS.md 的"关键文件"表格

**修改现有技术方案：**
1. 更新 `技术方案文档.md` 中的技术选型说明
2. 检查 `架构设计详解.md` 是否需要调整
3. 验证代码示例是否仍然有效
4. 更新相关的协议规范（如适用）

**协议规范更新：**
1. 在 `PROTOCOL_SPEC.md` 中更新消息结构
2. 添加新的消息类型示例
3. 更新实现指南中的代码示例
4. 记录版本号和向后兼容性说明
5. 在 `架构设计详解.md` 中更新通信协议描述

**架构优化：**
1. 在 `架构设计详解.md` 中提出优化方案
2. 评估对技术方案的影响（`技术方案文档.md`）
3. 更新组件职责和通信协议
4. 添加或更新代码示例

## 依赖关系

```
技术方案文档.md (技术选型基础)
    ↓
架构设计详解.md (架构实现)
    ↓
PROTOCOL_SPEC.md (协议规范)
    ↓
代码实现 (PC Agent, AI-Gateway, Skill Server)
```

**关键约束：**
- 代码实现应严格遵循 `架构设计详解.md` 中的设计
- 所有 WebSocket 消息应符合 `PROTOCOL_SPEC.md` 中的规范
- 技术选型应基于 `技术方案文档.md` 中的理由
- 新增功能时同步更新相关技术文档
