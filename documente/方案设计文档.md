## 一、方案概述

### 1.1 项目背景

现有企业 IM 工具（DingTalk/Feishu/Slack）的聊天消息仅支持纯文本，无法承载 AI 编码工具的富文本输出（代码块、Markdown、流式响应、Tool Use 等）。

**V2 方案**：通过 IM 已有的 "/" 命令框架触发 **Skill 交互**，用户选择 SKILL 后在 Skill 小程序中与 AI 工具进行多轮对话，对话结果可选中发送回 IM 聊天。

### 1.2 方案目标

| 阶段 | 目标 | 周期 |
|------|------|------|
| **Demo** | 验证核心技术链路可行性，开发本地联调 Demo | 8 周 |
| **最终实现** | 完成 IM 平台集成，支持生产环境部署 | 12 周 |

## 二、Demo 阶段方案

### 2.1 系统边界

#### 我们构建的组件

| 组件 | 性质 | 运行位置 | 职责 |
|------|------|----------|------|
| **PCAgent** | 独立 Node.js 进程 | 开发者 PC | 桥接本地 OpenCode ↔ AI-Gateway |
| **AI-Gateway** | Spring Boot 服务 | 服务端 / 本地 | 连接管理、AK/SK 认证、透传中继 |
| **Skill Server** | Spring Boot 服务 | 服务端 / 本地 | 会话持久化、消息存储、客户端推流 |
| **Web UI** | React Web 应用 | 浏览器 | 协议适配、富文本渲染、用户交互 |
| **AK/SK 管理后台** | Spring Boot + React | 服务端 / 本地 | 用户管理、密钥管理 |

#### 不在边界内

- **OpenCode**：第三方 AI 编码工具，PCAgent 通过 SDK 连接
- **MySQL**：基础设施，使用现有实例或 Docker 启动
- **Redis**：基础设施，使用现有实例或 Docker 启动

### 2.2 整体架构

```
┌─────────────────────────────────────────────────────────┐
│  OpenCode (本地, localhost:54321)                         │
└────────────────────────┬────────────────────────────────┘
                         │ @opencode-ai/sdk
                         ▼
┌─────────────────────────────────────────────────────────┐
│  PCAgent (独立 Node.js 进程)                              │
│  事件订阅 + WebSocket 连接 + AK/SK 签名 + 心跳重连        │
└────────────────────────┬────────────────────────────────┘
                         │ WebSocket (AK/SK)
                         ▼
┌─────────────────────────────────────────────────────────┐
│  AI-Gateway (:8081)  ←──Redis Pub/Sub──→  多实例协调     │
└────────────────────────┬────────────────────────────────┘
                         │ WebSocket (internal token)
                         ▼
┌─────────────────────────────────────────────────────────┐
│  Skill Server (:8082)  ←──MySQL──→  消息持久化            │
└──────────┬─────────────────────────────┬────────────────┘
           │ WebSocket 推流              │ REST API
           ▼                             ▼
┌──────────────────────┐   ┌──────────────────────────────┐
│ Web UI (:3000)        │   │ AK/SK 管理后台 (:3001/:8083) │
│ React + 协议适配      │   │ 用户注册 + 密钥管理           │
└──────────────────────┘   └──────────────────────────────┘
```

**核心数据流**：OpenCode 原始事件从底层一路透传到客户端，中间各层不做协议解析或转换。客户端作为协议适配层，负责将 OpenCode 事件解析为可渲染的 UI 内容。

### 2.3 用户交互流程（Demo）

```
1. 用户打开 AK/SK 管理后台 (http://localhost:3001)
   → 注册账号 → 登录 → 创建 AK/SK

2. 用户启动 PCAgent
   → npm start -- --ak=xxx --sk=xxx --gateway=ws://localhost:8081/ws/agent
   → PCAgent 连接 AI-Gateway → 显示"已连接"

3. 用户打开 Web UI (http://localhost:3000)
   → 看到会话列表（左侧）和对话区域（右侧）

4. 用户创建新会话或选择已有会话

5. 用户在输入框输入问题，点击发送
   → 消息通过 REST API 发送到 Skill Server
   → Skill Server 持久化 → 转发到 AI-Gateway → 转发到 PCAgent
   → PCAgent 调用 OpenCode API

6. OpenCode 流式响应
   → SSE 事件流 → PCAgent → AI-Gateway → Skill Server → WebSocket 推流
   → Web UI 逐字渲染（Markdown + 代码高亮 + Tool Use 卡片）

7. 用户关闭 Web UI，稍后重新打开
   → 会话列表恢复 → 点击会话 → 历史消息从数据库加载

8. 用户在管理后台管理 AK/SK
   → 查看列表 → 创建新的 → 删除不需要的 → 禁用/启用
```

### 2.4 核心技术实现

#### 2.4.1 透传模式

**核心原则**：服务端不解析 OpenCode 事件，保持协议独立性。

```
OpenCode 事件 (原始 JSON)
    ↓
PCAgent event.subscribe() 接收
    ↓ WebSocket 透传
AI-Gateway EventRelayService
    ↓ WebSocket 透传
Skill Server GatewayRelayService
    ↓ WebSocket 推流
客户端 protocol/ 模块解析
    ↓ 类型化事件
UI 渲染
```

**优势**：
- 服务端无需关心 OpenCode 版本更新
- 协议适配集中在客户端，易于维护
- 支持未来多工具扩展（新增工具仅需新增客户端解析器）

**Gateway 连接管理**：

```typescript
class GatewayConnection {
  private ws: WebSocket | null = null;
  private retryCount = 0;
  private maxRetries = 3;
  private baseDelay = 1000; // 1s, 2s, 4s

  async connect(url: string, authParams: AuthParams): Promise<void> {
    const fullUrl = `${url}?ak=${authParams.ak}&ts=${authParams.ts}&nonce=${authParams.nonce}&sign=${authParams.sign}`;
    this.ws = new WebSocket(fullUrl);

    this.ws.on('open', () => {
      console.log('[PCAgent] Connected to AI-Gateway');
      this.retryCount = 0;
      this.startHeartbeat();
    });

    this.ws.on('close', () => {
      this.stopHeartbeat();
      this.reconnect(url, authParams);
    });
  }

  private async reconnect(url: string, authParams: AuthParams): Promise<void> {
    if (this.retryCount >= this.maxRetries) {
      console.error('[PCAgent] Max retries reached');
      return;
    }
    const delay = this.baseDelay * Math.pow(2, this.retryCount);
    this.retryCount++;
    console.log(`[PCAgent] Reconnecting in ${delay}ms (attempt ${this.retryCount})`);
    await sleep(delay);
    await this.connect(url, AkSkAuth.sign(authParams.ak, authParams.sk));
  }
}
```

**AK/SK 签名生成**：

```typescript
import * as crypto from 'crypto';

class AkSkAuth {
  static sign(ak: string, sk: string): AuthParams {
    const ts = Date.now().toString();
    const nonce = crypto.randomUUID();
    const message = `${ak}\n${ts}\n${nonce}`;
    const sign = crypto.createHmac('sha256', sk).update(message).digest('hex');
    return { ak, ts, nonce, sign };
  }
}
```

**WebSocket 连接管理**：

```java
@Component
public class AgentWebSocketHandler extends TextWebSocketHandler {

    @Override
    public void afterConnectionEstablished(WebSocketSession session) {
        // 1. 从握手参数提取 AK/SK 签名
        String ak = getQueryParam(session, "ak");
        String ts = getQueryParam(session, "ts");
        String nonce = getQueryParam(session, "nonce");
        String sign = getQueryParam(session, "sign");

        // 2. 验证签名
        if (!akSkAuthService.verify(ak, ts, nonce, sign)) {
            session.close(CloseStatus.POLICY_VIOLATION);
            return;
        }

        // 3. 注册 Agent
        Long userId = akSkAuthService.getUserIdByAk(ak);
        agentRegistryService.register(session, userId, ak);
    }

    @Override
    protected void handleTextMessage(WebSocketSession session, TextMessage message) {
        GatewayMessage msg = objectMapper.readValue(message.getPayload(), GatewayMessage.class);

        switch (msg.getType()) {
            case "register":
                agentRegistryService.updateInfo(session, msg);
                break;
            case "heartbeat":
                agentRegistryService.refreshHeartbeat(session);
                break;
            case "tool_event":
            case "tool_done":
            case "tool_error":
            case "session_created":
                // 透传到 Skill Server
                eventRelayService.relayToSkillServer(session, msg);
                break;
        }
    }
}
```

**AK/SK 验证服务**：

```java
@Service
public class AkSkAuthService {

    @Autowired private RedisTemplate<String, String> redisTemplate;
    @Autowired private AccessKeyMapper accessKeyMapper;

    public boolean verify(String ak, String ts, String nonce, String sign) {
        // 1. 检查时间戳窗口 (±5 分钟)
        long timestamp = Long.parseLong(ts);
        long now = System.currentTimeMillis();
        if (Math.abs(now - timestamp) > 300_000) {
            return false;
        }

        // 2. 检查 Nonce 防重放 (Redis SET NX, TTL 5 分钟)
        Boolean isNew = redisTemplate.opsForValue()
            .setIfAbsent("nonce:" + nonce, "1", Duration.ofMinutes(5));
        if (!Boolean.TRUE.equals(isNew)) {
            return false;
        }

        // 3. 查询 SK
        AccessKey accessKey = accessKeyMapper.findByAccessKey(ak);
        if (accessKey == null || !"ACTIVE".equals(accessKey.getStatus())) {
            return false;
        }

        // 4. 计算期望签名并比对 (常数时间比较)
        String expected = HmacUtils.hmacSha256Hex(
            accessKey.getSecretKey(), ak + "\n" + ts + "\n" + nonce);
        return MessageDigest.isEqual(
            expected.getBytes(), sign.getBytes());
    }
}
```

```java
    // 本地 Agent 连接池
    private final ConcurrentMap<String, WebSocketSession> agentSessions = new ConcurrentHashMap<>();

    // 透传到 Skill Server: 不解析消息内容，只附加 agentId
    public void relayToSkillServer(WebSocketSession session, GatewayMessage msg) {
        String agentId = getAgentId(session);
        GatewayMessage forwarded = msg.withAgentId(agentId);
        skillServerWSClient.send(forwarded);
    }

    // 从 Skill Server 接收 invoke, 路由到正确的 Agent
    public void routeToAgent(String agentId, GatewayMessage msg) {
        WebSocketSession local = agentSessions.get(agentId);
        if (local != null && local.isOpen()) {
            local.sendMessage(new TextMessage(toJson(msg)));
        } else {
            // 跨实例路由: 通过 Redis Pub/Sub
            redisMessageBroker.publish("agent:" + agentId, toJson(msg));
        }
    }
}
```

```java
public class SkillSessionService {

    public SkillSession createSession(Long userId, Long skillDefId) {
        // 检查并发限制
        int activeCount = sessionMapper.countActiveByUserId(userId);
        if (activeCount >= 10) {
            throw new BusinessException(429, "活跃会话数已达上限 (10)");
        }

        SkillSession session = new SkillSession();
        session.setUserId(userId);
        session.setSkillDefinitionId(skillDefId);

        session.setLastActiveAt(LocalDateTime.now());

        return session;

    public void sendMessage(Long sessionId, String content) {
        SkillSession session = sessionMapper.selectById(sessionId);

        // 1. 持久化用户消息
        SkillMessage msg = new SkillMessage();
        msg.setSessionId(sessionId);
        msg.setSeq(getNextSeq(sessionId));
        msg.setRole("USER");
        msg.setContent(content);
        messageMapper.insert(msg);

        sessionMapper.touchSession(sessionId);

        // 3. 发送 invoke 到 Gateway → PCAgent → OpenCode
        gatewayRelayService.sendInvoke(session.getAgentId(), sessionId, "chat",
            Map.of("text", content, "toolSessionId", session.getToolSessionId()));
    }
```

#### 2.4.5 Web UI 实现

**WebSocket 连接管理**：

```typescript
class WebSocketService {
  private ws: WebSocket | null = null;
  private listeners: Map<string, Function[]> = new Map();

  connect(sessionId: string): void {
    const url = `ws://localhost:8082/ws/skill/stream/${sessionId}`;
    this.ws = new WebSocket(url);

    this.ws.onmessage = (event) => {
      const msg = JSON.parse(event.data);
      this.emit(msg.type, msg);
    };

    this.ws.onclose = () => {
      // 自动重连 (最多 3 次, 指数退避)
      this.reconnect(sessionId);
    };
  }

  on(type: string, callback: Function): void {
    if (!this.listeners.has(type)) this.listeners.set(type, []);
    this.listeners.get(type)!.push(callback);
  }

  private emit(type: string, data: any): void {
    this.listeners.get(type)?.forEach(fn => fn(data));
  }
}
```

```typescript
// protocol/OpenCodeEventParser.ts
export class OpenCodeEventParser {
  parse(rawEvent: any): ParsedEvent {
    switch (rawEvent.type) {
      case 'content_block_delta':
        return { type: 'text_delta', text: rawEvent.delta.text };
      case 'tool_use':
        return {
          type: 'tool_use',
          toolName: rawEvent.name,
          input: rawEvent.input,
          output: null // 等待 tool_result
        };
      case 'tool_result':
        return { type: 'tool_result', toolUseId: rawEvent.tool_use_id, output: rawEvent.content };
      case 'message_stop':
        return { type: 'done' };
      default:
        return { type: 'unknown', raw: rawEvent };
    }
  }
}

// protocol/StreamAssembler.ts
export class StreamAssembler {
  private buffer: string = '';

  append(delta: string): string {
    this.buffer += delta;
    return this.buffer;
  }

  reset(): void {
    this.buffer = '';
  }

  getContent(): string {
    return this.buffer;
  }
}
```

**useSkillStream Hook**：

```typescript
function useSkillStream(sessionId: string) {
  const [messages, setMessages] = useState<Message[]>([]);
  const [isStreaming, setIsStreaming] = useState(false);
  const wsRef = useRef<WebSocketService | null>(null);
  const assemblerRef = useRef(new StreamAssembler());
  const parserRef = useRef(new OpenCodeEventParser());

  useEffect(() => {
    const ws = new WebSocketService();
    wsRef.current = ws;
    ws.connect(sessionId);

    ws.on('delta', (msg: any) => {
      setIsStreaming(true);
      const parsed = parserRef.current.parse(msg.content);
      if (parsed.type === 'text_delta') {
        const full = assemblerRef.current.append(parsed.text);
        setMessages(prev => updateLastAssistant(prev, full));
      } else if (parsed.type === 'tool_use') {
        setMessages(prev => [...prev, { role: 'tool', toolUse: parsed }]);
      }
    });

    ws.on('done', () => {
      setIsStreaming(false);
      assemblerRef.current.reset();
    });

    ws.on('agent_offline', () => {
      // 显示离线状态
    });

    return () => ws.disconnect();
  }, [sessionId]);

  const sendMessage = async (text: string) => {
    setMessages(prev => [...prev, { role: 'user', content: text }]);
    await fetch(`/api/skill/sessions/${sessionId}/messages`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ content: text, role: 'USER' })
    });
  };

  return { messages, isStreaming, sendMessage };
}
```

## 三、最终实现阶段方案

### 3.1 系统边界

| 组件 | 性质 | 运行位置 | 职责 |
|------|------|----------|------|
| **PCAgent Plugin** | Electron Main Process 模块 | IM 客户端 | 桥接 OpenCode ↔ AI-Gateway |
| **AI-Gateway** | Spring Boot 集群 | 服务端 | 连接管理、AK/SK 认证、透传中继 |
| **Skill Server** | Spring Boot 集群 | 服务端 | 会话持久化、推流、IM 消息发送 |
| **Skill 小程序** | IM 小程序 | IM 客户端 | "/" 触发、Mini Bar、对话 UI |

**不在边界内**：IM 客户端 (已有)、IM 服务端 (已有)、"/" 命令框架 (已有)、OpenCode (第三方)

### 3.2 用户交互流程（最终）

```
1. 用户在 IM 输入框输入 "/"
   → 展示 SKILL 列表 → 选择 "OpenCode"

2. IM 聊天框出现高亮 SKILL 行: "✦ OpenCode"
   → 用户输入问题 → 点击"生成"

3. IM 聊天区出现 Mini Bar: [🔄 OpenCode 处理中… ▢ 展开]

4. 用户点击"展开" → Skill 小程序覆盖 IM 聊天窗
   → 流式渲染 AI 回复 (Markdown + 代码高亮 + Tool Use)

5. 用户可继续多轮对话

6. 用户选中 AI 回复文本 → 浮现"发送到聊天"按钮
   → 点击 → IM 聊天窗显示该消息

7. 用户点击"收起" → Skill 小程序收起 → Mini Bar 仍显示
```

### 3.3 与 Demo 的差异和改造点

| 组件 | Demo | 最终 | 改造工作 |
|------|------|------|----------|
| PCAgent | 独立进程 | IM 插件 | 适配 Electron 生命周期，独立子进程运行 |
| 客户端 | React Web | IM 小程序 | 适配容器 API，新增 MiniBar + 选文发送 |
| 触发 | URL 直接访问 | "/" 命令 | 注册 SKILL，响应回调 |
| IM 消息 | 空实现 | 实际调用 | 对接 IM 平台 API |

#### 选文发送到 IM

```tsx
const SendToImButton: React.FC = () => {
  const [selection, setSelection] = useState<string>('');
  const [position, setPosition] = useState<{ x: number; y: number } | null>(null);

  useEffect(() => {
    const handleSelectionChange = () => {
      const sel = window.getSelection()?.toString();
      if (sel && sel.length > 0) {
        const range = window.getSelection()!.getRangeAt(0);
        const rect = range.getBoundingClientRect();
        setSelection(sel);
        setPosition({ x: rect.left, y: rect.top - 40 });
      } else {
        setPosition(null);
      }
    };
    document.addEventListener('selectionchange', handleSelectionChange);
    return () => document.removeEventListener('selectionchange', handleSelectionChange);
  }, []);

  const sendToIm = async () => {
    await fetch(`/api/skill/sessions/${sessionId}/send-to-im`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ content: selection, chatId: imChatId })
    });
    setPosition(null);
  };

  if (!position) return null;

  return (
    <button
      style={{ position: 'fixed', left: position.x, top: position.y }}
      onClick={sendToIm}
      className="bg-blue-500 text-white px-3 py-1 rounded shadow"
    >
      发送到聊天
    </button>
  );
};
```

### 3.5 多 AI 工具支持方案

```
            ▼              ▼              ▼
       [GW Node 1]   [GW Node 2]   [GW Node 3]
```

**故障恢复**：

| 故障 | 恢复策略 |
|------|----------|
| PCAgent 断连 | 指数退避重连 (1s→2s→4s), 最多 3 次 |
| Gateway 宕机 | LB 自动切换, PCAgent 重连 |
| Skill Server 宕机 | LB 切换, 无状态即时恢复 |
| Redis 不可用 | 降级为本地路由 (单实例模式) |
| MySQL 不可用 | 推流不受影响, 持久化返回错误 |

### 3.7 性能优化方案

| 优化项 | 方案 | 目标 |
|--------|------|------|
| 连接池 | HikariCP max=20, Redis max-active=50 | 减少连接开销 |
| 流式处理 | delta 事件实时推送，不等待完整消息 | 首字延迟 < 100ms |
| 数据库索引 | user_id, session_id, seq 组合索引 | 查询 < 50ms |
| 前端渲染 | 虚拟滚动 (react-window), 懒加载 | 1000 条消息不卡顿 |
| 消息分页 | 每次 50 条，按 seq 排序 | 加载 < 500ms |

### 3.8 安全方案

| 层面 | 措施 |
|------|------|
| 传输 | WSS/HTTPS, TLS 1.2+ |
| 认证 | AK/SK HMAC-SHA256, JWT, Internal Token |
| 防护 | 时间戳防重放, Nonce 防重复, 常数时间比较 |
| 数据 | BCrypt 密码加密, 参数化查询, escapeHtml |
| 访问 | 用户隔离 (user_id), 单设备登录, 并发限制 (10) |
