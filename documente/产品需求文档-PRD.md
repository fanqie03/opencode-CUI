## 一、产品概述

### 1.1 产品定位

OpenCode-CUI 是一个分布式事件驱动架构的 AI 编码工具集成方案，旨在将本地 AI 编码工具（如 OpenCode）无缝集成到企业 IM 平台中，让开发者无需离开日常沟通工具即可使用 AI 辅助编码。

### 1.2 目标用户

- **主要用户**: 使用企业 IM 工具（DingTalk/Feishu/Slack）的开发者
- **次要用户**: 开发团队的技术负责人、架构师
- **潜在用户**: 需要 AI 编码辅助的所有技术人员

### 1.3 核心价值

1. **无缝集成**: 在 IM 中直接使用 AI 编码工具，无需切换应用
2. **协作增强**: 可以将 AI 生成的代码和解决方案直接分享到团队聊天
3. **隐私保护**: 本地模式确保代码不离开开发者的机器
4. **可扩展性**: 支持多种 AI 工具（OpenCode、Claude Code、OpenClaw）

## 三、Demo 阶段需求详情

### 3.1 功能需求

#### FR-D1: PCAgent 独立运行
**优先级**: P0（必须）

**描述**: PCAgent 作为独立的 Node.js 进程运行，不依赖 IM 客户端。

**功能点**:
- 支持命令行启动：`npm start -- --ak=xxx --sk=xxx --gateway=ws://localhost:8080`
- 支持配置文件：`config.json` 或环境变量
- 订阅 OpenCode 的所有事件类型
- 通过 WebSocket 连接到 AI-Gateway
- 实现 AK/SK 签名认证
- 实现心跳保活和自动重连

**验收标准**:
- ✅ PCAgent 能够成功启动并连接到 AI-Gateway
- ✅ 能够订阅 OpenCode 事件并实时转发
- ✅ 连接断开后能够自动重连（最多 3 次）
- ✅ 提供清晰的日志输出和错误诊断

---

#### FR-D2: Web UI 基础功能
**优先级**: P0（必须）

**描述**: 使用 React 开发的 Web 应用，模拟 IM 客户端的交互界面。

**功能点**:
- 通过 WebSocket 连接到 Skill Server
- 发送消息到 OpenCode
- 接收并显示 OpenCode 的回复
- 支持 Markdown 渲染
- 支持代码语法高亮
- 支持流式渲染（逐字显示）
- 支持 Tool Use 可视化

**验收标准**:
- ✅ 能够在浏览器中打开并正常使用
- ✅ 能够发送消息并接收回复
- ✅ Markdown 和代码块正确渲染
- ✅ 流式渲染无明显卡顿
- ✅ Tool Use 以卡片形式显示

---

#### FR-D3: 会话历史管理
**优先级**: P0（必须）

**描述**: 支持会话的持久化和恢复。

**功能点**:
- 左侧显示会话列表
- 点击会话加载历史消息
- 支持创建新会话
- 支持删除会话
- 关闭页面后重新打开能恢复完整对话

**验收标准**:
- ✅ 会话列表正确显示
- ✅ 历史消息完整恢复（包括代码块、Tool Use）
- ✅ 会话切换无明显延迟
- ✅ 删除会话后数据被清除

---

#### FR-D4: AK/SK 管理后台
**优先级**: P0（必须）

**描述**: 用户自助申请和管理 AK/SK 的后台系统。

**功能点**:
- 用户注册和登录
- 创建 AK/SK
- 查看 AK/SK 列表
- 删除 AK/SK
- 启用/禁用 AK/SK

**验收标准**:
- ✅ 用户可以注册和登录
- ✅ 可以创建、查看、删除 AK/SK
- ✅ 禁用的 AK/SK 无法通过认证
- ✅ UI 美观、易用

---

#### FR-D5: 多实例协调
**优先级**: P1（重要）

**描述**: 支持多个 AI-Gateway 和 Skill Server 实例的协调。

**功能点**:
- 使用 Redis Pub/Sub 实现消息路由
- 支持实例的动态上下线
- 支持故障自动切换

**验收标准**:
- ✅ 多个实例能够正确路由消息
- ✅ 实例故障时能够自动切换
- ✅ 消息不丢失、不乱序

---

#### FR-D6: 错误处理
**优先级**: P1（重要）

**描述**: 完善的错误处理和用户提示。

**功能点**:
- WebSocket 连接断开后自动重连
- OpenCode 离线时显示离线状态
- 离线消息排队
- 所有错误都有明确的提示信息

**验收标准**:
- ✅ 连接断开后自动重连
- ✅ OpenCode 离线时显示离线状态
- ✅ 离线消息在 OpenCode 上线后自动发送
- ✅ 错误提示清晰、友好

---
### 3.2 非功能需求

#### NFR-D1: 性能要求
- WebSocket 消息端到端延迟 p95 < 200ms
- 流式渲染无明显卡顿
- 会话历史加载时间 < 1s（100 条消息）
- 并发 10 个会话时系统稳定运行

#### NFR-D2: 可用性要求
- 系统可用性 > 95%（Demo 阶段）
- 故障恢复时间 < 5 分钟

#### NFR-D3: 安全要求
- AK/SK 使用 HMAC-SHA256 签名
- 密码使用 BCrypt 加密
- WebSocket 支持 WSS（可选）
- 防止 XSS 攻击（Markdown 渲染时 escapeHtml）

#### NFR-D4: 可维护性要求
- 代码符合规范
- 关键逻辑有单元测试
- 提供完整的部署文档

## 九、附录

### 9.1 术语表

| 术语 | 说明 |
|------|------|
| PCAgent | 运行在用户本地的代理程序，负责订阅 OpenCode 事件并转发到 AI-Gateway |
| AI-Gateway | WebSocket 连接管理服务，负责 AK/SK 验证和消息路由 |
| Skill Server | 业务逻辑服务，负责消息持久化、会话管理、与 AI 工具交互 |
| Skill 小程序 | 运行在 IM 客户端小程序容器中的前端应用 |
| Mini Bar | IM 聊天框上方的微型状态条，显示 AI 工具的状态 |
| 透明中继 | 服务端不解析 OpenCode 事件，协议适配在客户端完成 |
| Tool Use | AI 工具调用外部工具（如文件读写、终端命令）的过程 |
