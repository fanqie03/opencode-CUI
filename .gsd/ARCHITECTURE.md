## Overview

**opencode-CUI** is a Skill-based conversation integration solution that bridges IM platforms (DingTalk, Feishu, Slack) with local OpenCode AI coding instances. Users trigger Skill interactions via "/" commands in their IM client, which opens a mini-app for multi-turn AI coding conversations. Responses can be selectively sent back to the IM chat.

The system follows a **transparent pass-through** architecture — backend services relay OpenCode events without parsing or transforming them. Protocol interpretation happens exclusively in the frontend mini-app.

## Data Flow

1. **User triggers "/" command** in IM client → Skill Mini-App opens
2. **Mini-App connects** to Skill Server via WebSocket (`/ws/client`)
3. **User sends message** → Skill Server persists to DB → forwards to AI-Gateway via WebSocket/Redis
4. **AI-Gateway routes** the request to the correct PC Agent via WebSocket (`/ws/agent`)
5. **PC Agent relays** to local OpenCode instance via `@opencode-ai/sdk`
6. **OpenCode streams response** → PC Agent relays upstream → AI-Gateway relays via Redis + WebSocket
7. **Skill Server receives** stream → persists messages → streams to Mini-App via WebSocket
8. **Mini-App renders** the response (Markdown, code blocks, syntax highlighting)
9. **(Optional) User clicks "Send to Chat"** → Skill Server dispatches formatted message to IM chat via REST API
